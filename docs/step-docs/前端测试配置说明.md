# 前端测试配置说明文档

## 概述
本文档记录了预约系统前端应用的测试策略、测试环境、测试用例和测试结果，确保所有业务功能都能正常运行。本文档涵盖手动测试、单元测试和集成测试的详细信息。

## 测试环境

### 开发环境
- 操作系统: Linux
- Node.js版本: v20.x
- 浏览器: Chrome 120+
- 应用地址: http://localhost:3000

### 测试框架
- 测试框架: Jest 29.7.0
- 测试工具: React Testing Library 14.0.0
- DOM测试: @testing-library/jest-dom 6.8.0
- 集成测试: TestContainers (Node.js环境需要polyfill)
- 状态管理测试: redux-mock-store 1.5.5

## 测试策略

### 1. 测试分层

- **单元测试**: 针对独立组件、函数和服务进行测试
- **集成测试**: 测试组件间的交互和API服务调用，使用TestContainers模拟后端环境
- **手动测试**: 针对UI/UX和复杂场景进行人工验证

### 2. 测试文件结构

```
__tests__/
  components/
    pages/
      BookingPage.test.js      # 预约页面组件测试
      LoginPage.test.js        # 登录页面组件测试
  services/
    authService.test.js        # 认证服务测试
    bookingService.test.js     # 预约服务测试
  store/
    bookingSlice.test.js       # 预约状态管理测试
    userSlice.test.js          # 用户状态管理测试
  utils/
    mockStore.js               # Redux模拟存储工具
    testUtils.js               # 测试工具函数
  integration/
    bookingService.integration.test.ts  # TestContainers集成测试
    openapi.yml                # OpenAPI规范文件
```

## 单元测试详情

### 1. 页面组件测试

**BookingPage.test.js**: 测试预约页面的核心功能，包括日期选择、时间段选择、预约创建和取消等功能。

**LoginPage.test.js**: 测试登录页面的核心功能，包括发送验证码、验证验证码、登录成功后的重定向等功能。

### 2. 服务测试

**authService.test.js**: 测试认证服务的各种API调用，包括发送验证码、验证验证码、获取用户信息、登出等功能。

**bookingService.test.js**: 测试预约服务的各种API调用，包括获取预约列表、获取可用时间段、创建预约、取消预约等功能。

### 3. Redux状态管理测试

**bookingSlice.test.js**: 测试预约相关的Redux状态管理，包括预约列表、可用时间段、选中日期和时间段的状态管理，确保所有API调用路径使用`/appointments`而非`/bookings`。

**userSlice.test.js**: 测试用户相关的Redux状态管理，包括验证码发送、验证、倒计时、用户登录状态等。

## TestContainers集成测试

### 1. 测试概述

集成测试使用TestContainers启动一个模拟的API服务器容器，测试前端应用与后端服务的交互能力。使用`stoplight/prism`容器基于OpenAPI规范自动生成模拟响应。

### 2. 测试文件

**bookingService.integration.test.ts**: 测试完整的预约服务调用流程，包括API调用、错误处理、认证和授权等功能。

**openapi.yml**: 定义API规范，用于TestContainers模拟服务器生成响应。

### 3. 测试配置

- 使用`GenericContainer`启动Docker容器
- 映射容器端口到主机端口
- 挂载OpenAPI规范文件到容器中
- 动态配置API基础URL

### 4. Jest环境配置

**jest.config.js配置**:
- `testEnvironment: 'jsdom'` - 使用jsdom环境模拟浏览器
- `setupFilesAfterEnv: ['<rootDir>/jest.setup.js']` - 加载测试环境设置
- `moduleNameMapper` - 配置路径别名和CSS文件代理
- `testPathIgnorePatterns` - 排除特定测试文件
- `transform` - 使用babel-jest处理JS/TS文件

**jest.setup.js配置**:
- 导入`@testing-library/jest-dom`扩展匹配器
- 添加`TextEncoder`/`TextDecoder` polyfill
- 添加`ReadableStream` polyfill (集成测试需要)
- 添加`MessagePort` polyfill (集成测试需要)

## 已知问题

### 1. Hydration警告
在页面加载时出现以下警告：
```
A tree hydrated but some attributes of the server rendered HTML didn't match the client properties.
```

#### 影响
此警告不会影响功能，仅是开发时的警告信息。

#### 解决方案
这是一个常见的Next.js hydration警告，通常由客户端和服务端渲染的随机ID不匹配导致。在生产环境中不会出现此问题。
