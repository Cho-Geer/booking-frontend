# 预约系统前后端接口设计

## 概述
本文档基于预约系统的技术栈、前端架构、数据模型和UI设计系统，设计完整的前后端接口规范。接口设计遵循RESTful原则，采用现代化的API设计模式，支持原子化前端架构和状态管理需求，确保与数据模型设计、业务功能模块和项目需求文档保持一致性。

## 更新记录
- **v2.1.0**: 基于数据模型设计和业务功能模块进行接口优化，增加设计系统接口支持，统一技术栈版本规范

## 设计原则

### 1. RESTful设计原则
- **资源导向**: 以资源为中心设计URL
- **状态无关**: 每个请求包含完整的信息
- **统一接口**: 统一的HTTP方法和状态码
- **分层系统**: 支持缓存和代理服务器

### 2. 安全性原则
- **JWT认证**: 使用JWT进行身份验证
- **HTTPS**: 所有接口使用HTTPS协议
- **输入验证**: 严格的参数验证和清洗
- **限流保护**: 防止API滥用

### 3. 性能优化
- **分页**: 列表接口支持分页，默认10条/页，最大100条/页
- **缓存**: 合理使用HTTP缓存，支持Redis热点数据缓存
- **压缩**: 支持Gzip压缩，优化数据传输
- **字段过滤**: 支持选择返回字段，减少不必要数据传输
- **数据库优化**: 索引优化、查询优化、连接池管理

## 接口规范

### 基础信息
- **Base URL**: `https://api.booking-system.com/api/v1`
- **认证方式**: Bearer Token (JWT)
- **数据格式**: JSON
- **字符编码**: UTF-8
- **时间格式**: ISO 8601 (YYYY-MM-DDTHH:mm:ss.sssZ)

## 用户认证接口

### 1. 发送验证码
**POST** `/auth/send-code`

**请求体**:
```typescript
interface SendCodeRequest {
  phone: string; // 手机号
  type: 'login' | 'register' | 'reset';
}
```

**响应体**:
```typescript
interface SendCodeResponse {
  success: boolean;
  expiresIn: number; // 验证码有效期（秒）
  remainingAttempts: number; // 剩余尝试次数
}
```

**业务规则**:
- 同一手机号60秒内只能发送一次验证码
- 验证码有效期5分钟
- 同一手机号每天最多发送10次验证码
- 支持登录、注册、重置密码、修改手机号等场景
