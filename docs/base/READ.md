# 预约系统网站需求文档

## 1. 项目概述
本项目旨在开发一个现代化、用户友好的线上预约系统，提供会议室预约、活动预约和个人服务预约等功能，使用React、TypeScript和Tailwind CSS构建。

### 1.1 背景
随着远程办公和混合办公模式的普及，高效的会议室和活动预约系统变得越来越重要。本系统旨在简化预约流程，提高资源利用率，并提供良好的用户体验。

### 1.2 目标
- 提供直观的预约界面
- 实现灵活的会议室、活动和个人服务预约功能
- 支持实时预约状态显示和冲突检测
- 确保跨设备的一致体验
- 提供深色和浅色主题切换功能，满足不同用户的视觉偏好

### 1.3 适用范围
适用于需要管理会议室、活动空间和个人服务预约的企业和组织。

## 2. 功能需求

### 2.1 核心功能
- **预约管理**：创建、查看、编辑和取消预约
- **资源管理**：管理可预约资源（会议室、活动空间等）
- **用户管理**：用户注册、登录、个人信息管理
- **搜索与筛选**：按日期、时间、资源类型等条件筛选
- **通知系统**：预约确认、提醒和变更通知
- **主题切换**：支持深色和浅色主题，提高用户体验

### 2.2 会议室预约功能
- 查看会议室占用情况
- 选择会议室和时间段
- 添加会议详情（标题、参与者、设备需求等）
- 查看预约历史
- 取消或修改预约

### 2.3 活动预约功能
- 创建和发布活动
- 活动详情展示
- 活动报名和管理
- 活动提醒

### 2.4 个人服务预约功能
- 服务提供者管理
- 服务时间安排
- 用户预约服务
- 服务评价

### 2.5 管理后台功能
- 用户角色管理
- 资源分配和管理
- 预约记录查询和统计
- 系统配置

## 3. 业务规则与约束

### 3.1 预约规则
- 预约时间段：可预约的最小单位为30分钟
- 预约提前期：可提前最多30天预约
- 预约时长限制：单次预约最长不超过8小时
- 预约冲突处理：系统自动检测并提示冲突

### 3.2 用户权限
- 普通用户：创建和管理自己的预约
- 管理员：管理所有预约和系统配置
- 资源管理员：管理特定资源

### 3.3 数据安全
- 用户密码加密存储
- 敏感数据传输加密
- 定期数据备份

### 7.3 组件化规范
项目严格遵循Brad Frost提出的原子化设计方法论，将UI组件分解为五个层次，并明确定义每个层次的职责和状态管理策略：

- **原子组件（Atoms）**：最小的UI单元，不可再分
  - 示例：Button、Card等基础UI单元
  - 实现文件：`/src/components/atoms/`
  - 状态管理：仅包含自身必要的UI状态，使用useState管理
  - 主题集成：通过props接收主题配置或使用React Context访问全局UI状态

- **分子组件（Molecules）**：由原子组成的简单UI组合
  - 示例：BookingForm（预约表单）等简单组合组件
  - 实现文件：`/src/components/molecules/`
  - 状态管理：本地状态使用useState，共享状态通过props传递

- **有机体组件（Organisms）**：相对复杂的UI组件，形成独立的功能单元
  - 示例：BookingCalendar（预约日历）、BookingList（预约列表）等功能单元
  - 实现文件：`/src/components/organisms/`
  - 状态管理：UI状态使用React Context，业务状态通过props注入或使用Redux hooks

- **模板组件（Templates）**：页面级组件，定义内容结构和布局
  - 示例：AppLayout等布局组件
  - 实现文件：`/src/components/templates/`
  - 状态管理：连接Redux获取业务数据，使用React Context获取UI状态

- **包装组件（Wrappers）**：用于功能增强和布局控制的特殊组件
  - 示例：PageWrapper等条件包装组件
  - 实现文件：`/src/components/wrappers/`
  - 功能：实现智能布局选择、权限控制等横切关注点

- **页面组件**：Next.js路由页面，直接位于`/src/pages/`目录
  - 职责：集成所需组件，实现完整页面功能
  - 状态管理：根据需要使用Redux Toolkit获取和管理业务状态，使用React Context获取UI状态
  - 数据流：页面组件作为数据入口点，通过props向下传递给子组件

### 7.4 状态管理
项目严格按照以下分工实现状态管理，确保状态管理的清晰性和一致性：

#### UI状态 (React Context)
- 实现文件：`/src/contexts/UIContext.tsx`
- 功能：
  - 主题模式：浅色/深色主题切换及配置
  - 界面交互状态：侧边栏展开/收起、模态框显示/隐藏
  - 全局UI配置：语言设置（zh-CN/en-US）、响应式布局适配
  - 临时UI状态：全局加载状态、通知管理
  - 面包屑导航状态管理
- 使用范围：主要用于原子、分子、模板组件，提供统一的UI体验

#### 业务状态 (Redux Toolkit)
- 实现文件：`/src/store/`
- 功能：
  - 预约数据：预约列表、可用时间段、预约操作状态
  - 用户数据：用户信息、认证状态、权限信息
  - 异步操作：
    - getBookings: 获取用户预约列表
    - getAvailableSlots: 获取指定日期的可用时间段
    - createBooking: 创建新预约
    - cancelBooking: 取消预约
- 使用范围：主要用于页面组件和有机体组件，管理业务数据和流程

#### 数据获取 (TanStack Query)
- 实现文件：`/src/services/reactQuery.ts`及相关服务文件
- 功能：处理异步数据获取、缓存、同步和更新
- 集成方式：与Redux Toolkit配合使用，TanStack Query负责数据获取和缓存，Redux负责管理应用状态

#### 本地组件状态 (React useState)
- 适用场景：组件内部独立状态，如表单输入、展开/折叠等临时状态
- 使用原则：仅用于不影响其他组件的局部状态，避免组件间状态冗余

#### Redux Toolkit 配置

#### 行高
- **标题**：1.2-1.4
- **正文**：1.6

#### 字体应用规范
- 标题使用粗体或半粗体，确保视觉层次
- 正文使用常规字重，保证长时间阅读舒适度
- 辅助文本使用较小字体和次要文字颜色
- 等宽字体用于代码块、数据表格等需要对齐的内容
- **主题切换时字体样式规则**：
  - 字体大小、字重和行高在不同主题下保持一致
  - 字体颜色根据主题自动调整以确保最佳可读性
  - 深色主题使用浅色文字(#F8FAFC, #CBD5E1)，浅色主题使用深色文字(#0F172A, #334155)

### 8.4 间距系统

#### 基于8px网格系统
- **xs**：0.25rem (4px)
- **sm**：0.5rem (8px)
- **md**：1rem (16px)
- **lg**：1.5rem (24px)
- **xl**：2rem (32px)
- **2xl**：3rem (48px)
- **3xl**：4rem (64px)
- **4xl**：6rem (96px)
- **5xl**：8rem (128px)

#### 间距应用规范
- 使用一致的间距单位，避免随意使用像素值
- 组件内部使用较小间距（sm, md）
- 组件之间使用中等间距（md, lg）
- 区块之间使用较大间距（xl, 2xl）
- 确保内容区域有足够的留白，提高可读性

### 8.5 圆角系统
- **none**：0px (直角)
- **sm**：0.125rem (2px)
- **md**：0.375rem (6px)
- **lg**：0.5rem (8px)
- **xl**：0.75rem (12px)
- **2xl**：1.5rem (24px)
- **full**：9999px (圆形)

#### 圆角应用规范
- 按钮使用中等圆角（md）
- 卡片使用较大圆角（lg）
- 输入框使用中等圆角（md）
- 头像和圆形图标使用圆形（full）
- 表格和分割线使用直角（none）

### 8.6 阴影系统
- **sm**：0 1px 2px 0 rgba(0, 0, 0, 0.05)
- **md**：0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06)
- **lg**：0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05)
- **xl**：0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 10px 10px -5px rgba(0, 0, 0, 0.04)
- **2xl**：0 25px 50px -12px rgba(0, 0, 0, 0.25)

#### 阴影应用规范
- 基础组件使用轻微阴影（sm, md）
- 卡片和模态框使用中等阴影（md, lg）
- 下拉菜单和悬浮元素使用较深阴影（lg, xl）
- 避免过度使用阴影，保持界面简洁
- **主题切换时阴影规则**：
  - 深色主题使用标准阴影
  - 浅色主题阴影可以稍微加深以提高层次感

### 8.7 动画规范

#### 过渡时间
- **快速过渡**：0.15s
- **标准过渡**：0.2s
- **慢速过渡**：0.3s

#### 过渡效果
- **标准过渡**：all 0.2s ease-out
- **快速过渡**：all 0.15s ease-out
- **慢速过渡**：all 0.3s ease-in-out

#### 动画曲线
- **ease-out**：快速开始，缓慢结束，适合入场动画
- **ease-in-out**：缓慢开始和结束，适合对称动画
- **linear**：匀速动画，适合旋转等循环动画

#### 组件动画
- 按钮悬停：轻微放大效果 (scale: 1.02)
- 卡片悬停：轻微放大和阴影增强
- 模态框：淡入淡出和缩放效果
- 页面切换：淡入淡出和位移效果
- **主题切换动画**：平滑的颜色过渡，确保文本和背景色变化自然，避免视觉闪烁

#### 反馈动画
- 加载状态：旋转动画或脉冲效果
- 成功状态：淡入成功图标和文字
- 错误状态：抖动动画和错误提示
- 表单验证：实时反馈动画

### 8.8 组件状态
组件应包含以下状态，并提供明确的视觉反馈：

#### 交互状态
- **默认状态**：默认外观
- **悬停状态**：轻微亮度变化或阴影增强
- **激活状态**：颜色加深或添加内阴影
- **禁用状态**：透明度降低，鼠标样式改变
- **聚焦状态**：添加聚焦环或边框变化

#### 反馈状态
- **加载状态**：旋转动画或脉冲效果
- **成功状态**：绿色主题配成功图标
- **警告状态**：橙色主题配警告图标
- **错误状态**：红色主题配错误图标

### 8.11 主题切换机制

#### 实现方式
- 使用React Context API实现主题管理
- 支持基于类的Tailwind暗色模式切换
- 提供主题切换按钮，允许用户在深色和浅色主题之间切换
- 主题偏好保存在localStorage中，确保用户下次访问时保持一致的体验

#### 主题切换时的元素变化规则
1. **背景颜色**：
   - 深色主题：#0F172A (主背景), #1E293B (卡片背景)
   - 浅色主题：#F8FAFC (主背景), #F1F5F9 (卡片背景)

2. **字体颜色**：
   - 深色主题：主文字使用#F8FAFC，次要文字使用#CBD5E1
   - 浅色主题：主文字使用#0F172A，次要文字使用#334155
   - 确保在两种主题下文本与背景的对比度都符合WCAG 2.1 AA标准

3. **边框颜色**：
   - 深色主题：使用#334155
   - 浅色主题：使用#CBD5E1

4. **阴影效果**：
   - 深色主题：使用标准阴影
   - 浅色主题：阴影可以稍微加深以提高层次感

5. **主题切换动画**：
   - 应用平滑的颜色过渡效果，过渡时间约为0.3秒
   - 确保所有元素（特别是文本和背景）的颜色变化同步，避免视觉闪烁
