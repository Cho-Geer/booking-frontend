## 概述
本文档基于预约系统的功能需求、技术架构和UI设计系统，设计完整的数据模型。数据模型严格遵循UI设计系统规范，支持原子化前端架构（Atoms, Molecules, Organisms, Templates, Pages）和现代化的状态管理需求，确保与前端架构设计完全一致。

### 系统目标
- 提供直观的预约界面，支持会议室预约、活动预约和个人服务预约
- 实现灵活的资源预约功能，包括会议室、活动空间和个人服务
- 支持实时预约状态显示和冲突检测
- 确保跨设备的一致体验，支持深色和浅色主题切换
- 简化预约流程，提高资源利用率，提供良好的用户体验

### 技术栈要求（严格对齐前端架构设计）
- **前端框架**：Next.js 15.0.0+ (React 19.0.0+)
- **UI框架**：Tailwind CSS 3.4.0+ + Headless UI 2.0.0+
- **状态管理**：React Context (UI状态) + Redux Toolkit 2.0.0+ (业务状态)
- **数据获取**：TanStack Query 5.0.0+ (React Query)
- **表单验证**：React Hook Form 7.50.0+ + Zod 3.22.0+
- **日期选择**：React DayPicker 9.0.0+
- **动画库**：Framer Motion 11.0.0+
- **图标库**：Lucide React 0.300.0+ + Heroicons 2.0.0+
- **设计系统**：基于OKLCH色彩空间，支持主题切换机制

## 数据来源分析

### 1. 功能需求分析
- **会议室预约功能**：查看会议室占用情况、选择会议室和时间段、添加会议详情（标题、参与者、设备需求等）、查看预约历史、取消或修改预约
- **活动预约功能**：创建和发布活动、活动详情展示、活动报名和管理、活动提醒
- **个人服务预约功能**：服务提供者管理、服务时间安排、用户预约服务、服务评价
- **用户预约功能**：日期选择、时间段预约、预约状态管理
- **用户信息收集**：姓名、手机号、其他联系方式、咨询备注
- **管理员功能**：预约记录管理、时间段控制、统计报表、用户角色管理、资源分配和管理
- **系统功能**：验证码、通知、日志记录、主题切换（深色/浅色主题）

### 2. 技术架构要求（严格对齐前端架构设计）
- **前端架构**：原子化设计模式 (Atoms, Molecules, Organisms, Templates, Pages)
- **状态管理**：React Context (UI状态) + Redux Toolkit (业务状态) - 严格分离策略
- **数据获取**：TanStack Query 5.0.0+ (React Query) 提供强大的缓存和同步功能
- **表单处理**：React Hook Form 7.50.0+ + Zod 3.22.0+ 验证
- **设计系统**：支持设计Token管理，响应式设计断点系统，基于OKLCH色彩空间
- **组件架构**：支持组件状态管理（默认、悬停、激活、禁用、聚焦、加载、成功、警告、错误）
- **主题系统**：完整支持深色和浅色主题切换
- **性能优化**：组件懒加载、代码分割、数据缓存
- **无障碍设计**：支持ARIA属性、键盘导航、屏幕阅读器优化

### 3. UI设计系统约束（严格对齐设计规范）
- **色彩系统**：主色 #3B82F6，状态色彩（成功、警告、错误）- 支持OKLCH色彩空间
- **组件状态**：默认、悬停、激活、禁用、聚焦、加载、成功、警告、错误 - 完整状态管理
- **响应式设计**：移动端优先，断点系统 (sm:640px, md:768px, lg:1024px, xl:1280px)
- **设计Token**：颜色、间距、字体、圆角、阴影规范 - 支持CSS变量和主题切换
- **原子化设计**：支持Atoms、Molecules、Organisms、Templates、Pages五层架构
- **无障碍设计**：支持ARIA属性、键盘导航、屏幕阅读器优化

## 核心数据模型（严格对齐业务架构）

### 数据架构原则
- **业务状态与UI状态分离**：遵循前端架构设计的状态管理策略
- **原子化组件支持**：数据结构支持Atoms、Molecules、Organisms、Templates、Pages五层架构
- **设计Token兼容**：数据模型支持设计系统Token的动态配置
- **响应式数据支持**：数据结构支持移动端优先的响应式设计
- **无障碍数据支持**：数据模型包含ARIA属性和无障碍相关信息

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(100) NOT NULL,                    -- 用户姓名（支持原子化Input组件）
  phone VARCHAR(20) UNIQUE NOT NULL,            -- 手机号码（加密存储，支持原子化PhoneInput组件）
  phone_hash VARCHAR(64) UNIQUE NOT NULL,       -- 手机号哈希（用于查询，支持Molecules层数据验证）
  email VARCHAR(255),                             -- 邮箱（选填，支持原子化EmailInput组件）
  wechat VARCHAR(100),                          -- 微信号（选填，支持原子化Input组件）
  avatar VARCHAR(255),                          -- 头像URL（支持原子化Avatar组件）
  verification_code VARCHAR(6),                 -- 验证码（支持Molecules层VerificationForm）
  code_expires_at TIMESTAMP,                    -- 验证码过期时间（支持业务状态管理）
  is_verified BOOLEAN DEFAULT FALSE,            -- 是否已验证（支持UI状态与业务状态分离）
  user_type VARCHAR(20) DEFAULT 'customer',   -- 用户类型：customer, admin（支持权限管理Organisms）
  status VARCHAR(20) DEFAULT 'active',          -- 状态：active, inactive, blocked（支持UI组件状态）
  last_login_at TIMESTAMP,                      -- 最后登录时间（支持用户会话Molecules）
  login_count INTEGER DEFAULT 0,                 -- 登录次数（支持统计Organisms）
  ip_address INET,                               -- 注册IP地址（支持安全日志记录）
  user_agent TEXT,                              -- 用户代理（支持设备识别Molecules）
  theme_preference VARCHAR(20) DEFAULT 'light', -- 主题偏好：light, dark, auto（支持设计Token管理）
  language_preference VARCHAR(10) DEFAULT 'zh-CN', -- 语言偏好（支持国际化Templates）
  accessibility_enabled BOOLEAN DEFAULT FALSE,    -- 无障碍模式启用（支持ARIA属性管理）
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_users_phone_hash ON users(phone_hash);
```

#### 预约记录表 (appointments)
```sql
CREATE TABLE appointments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  appointment_number VARCHAR(20) UNIQUE NOT NULL, -- 预约编号 (AP-yyyymmdd-0001)
  user_id UUID REFERENCES users(id),              -- 用户ID
  service_id UUID REFERENCES services(id),        -- 服务ID
  appointment_date DATE NOT NULL,                 -- 预约日期
  time_slot_id UUID REFERENCES time_slots(id),    -- 时间段ID
  status VARCHAR(20) DEFAULT 'pending',           -- 状态：pending, confirmed, cancelled, completed
  
  -- 用户信息快照（预约时的用户信息）
  customer_name VARCHAR(100) NOT NULL,          -- 预约时姓名
  customer_phone VARCHAR(20) NOT NULL,          -- 预约时手机号（加密）
  customer_email VARCHAR(255),                   -- 预约时邮箱
  customer_wechat VARCHAR(100),                 -- 预约时微信
  notes TEXT,                                    -- 咨询内容备注
  
  -- 系统信息
  ip_address INET,                               -- 预约IP
  user_agent TEXT,                              -- 用户代理
  confirmation_sent BOOLEAN DEFAULT FALSE,      -- 是否发送确认通知
  reminder_sent BOOLEAN DEFAULT FALSE,          -- 是否发送提醒通知
  
  -- 时间记录
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  confirmed_at TIMESTAMP,                        -- 确认时间
  cancelled_at TIMESTAMP,                        -- 取消时间
  completed_at TIMESTAMP,                        -- 完成时间
  
  -- 约束
  UNIQUE(appointment_date, time_slot_id)        -- 同一天同一时间段只能有一个预约
);

-- 索引
CREATE INDEX idx_appointments_user_id ON appointments(user_id);
CREATE INDEX idx_appointments_date ON appointments(appointment_date);
CREATE INDEX idx_appointments_status ON appointments(status);
CREATE INDEX idx_appointments_created_at ON appointments(created_at);
CREATE INDEX idx_appointments_service_id ON appointments(service_id);
```

## 数据模型关系图

### 核心实体关系
- **User** (1) ──── (*) **Appointment**: 一个用户可以有多个预约
- **TimeSlot** (1) ──── (1) **Appointment**: 一个时间段只能有一个预约
- **ServiceCategory** (1) ──── (*) **Service**: 一个分类可以有多个服务
- **Service** (1) ──── (*) **Appointment**: 一个服务可以有多个预约
- **Appointment** (1) ──── (*) **Notification**: 一个预约可以有多个通知
- **User** (1) ──── (*) **Notification**: 一个用户可以有多个通知

### 索引优化
- 所有外键字段均建立索引
- 常用查询字段（如状态、日期）建立索引
- 高频过滤字段（如用户类型、服务分类）建立索引
