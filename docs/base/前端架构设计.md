## 1. 架构概述
本项目采用现代化前端架构，基于Next.js 15+、React 19+和Tailwind CSS 3.4+构建。架构设计遵循MCP（模块化组件化编程）原则，结合原子化设计模式，实现高内聚低耦合的组件系统，同时提供完整的深色/浅色主题切换支持。

### 1.1 架构优化目标
基于UI设计规范系统性优化方案，本次架构更新重点实现以下目标：
- **技术栈版本明确化**：明确技术栈具体版本要求，确保与UI设计规范一致
- **组件目录结构优化**：完善原子化设计目录结构，确保与UI设计规范组件分类对应
- **状态管理集成增强**：强化UI状态与业务状态的分离管理，支持主题切换机制
- **性能优化内容补充**：增加组件懒加载、数据缓存等性能优化策略
- **安全设计指导增强**：补充安全相关的架构设计考虑
- **国际化支持补充**：为未来国际化需求预留架构支持

### 1.1 架构原则
- **组件化优先**：将UI分解为可复用的原子、分子、有机体、模板和页面组件
- **状态分离**：UI状态（主题、交互）和业务状态（数据、逻辑）分离管理
- **TypeScript优先**：全面使用TypeScript确保类型安全
- **响应式设计**：移动端优先，适配各种屏幕尺寸
- **主题系统**：完整支持深色和浅色主题切换
- **性能优化**：组件懒加载、代码分割、数据缓存
- **可测试性**：组件设计考虑测试需求，便于单元测试和集成测试

### 1.2 技术栈（明确版本要求）
- **框架**：Next.js 15.0.0+ (React 19.0.0+)
- **UI库**：Tailwind CSS 3.4.0+ + Headless UI 2.0.0+
- **状态管理**：
  - UI状态：React Context (主题、界面交互状态)
  - 业务状态：Redux Toolkit 2.0.0+ (预约数据、用户数据)
- **数据获取**：TanStack Query (React Query) 5.0.0+
- **表单验证**：React Hook Form 7.50.0+ + Zod 3.22.0+
- **日期选择**：React DayPicker 9.0.0+
- **动画库**：Framer Motion 11.0.0+
- **图标库**：Lucide React 0.300.0+ + Heroicons 2.0.0+
- **构建工具**：Next.js内置构建系统 + Turbopack
- **设计系统**：基于OKLCH色彩空间，与UI设计规范完全对应

## 3. 组件化规范（严格对齐UI设计规范）
项目严格遵循Brad Frost提出的原子化设计方法论，将UI组件分解为五个层次，与UI设计规范的组件分类完全对应：

### 3.1 原子组件（Atoms）- 最小UI单元
- **定义**：最小的UI单元，不可再分，对应UI设计规范中的基础组件
- **实现文件**：`/src/components/atoms/`
- **状态管理**：使用React Context管理主题、颜色模式等UI状态
- **示例组件**：
  - Button：按钮组件，支持多种变体和主题切换
  - Input：输入框组件，支持深色/浅色主题
  - ThemeToggle：主题切换按钮
  - Card：卡片组件，支持不同主题下的背景和边框样式
  - Icons：图标组件，支持Lucide React和Heroicons
  - Label：标签组件，支持主题适配
  - Badge：徽章组件，支持多种状态显示
  - Avatar：头像组件，支持不同尺寸
  - Spinner：加载动画组件

**规范要求**：
- 严格遵循单一职责原则
- 支持完整的主题切换（light/dark模式）
- 完整的TypeScript类型定义
- 100%单元测试覆盖
- 支持所有UI设计规范定义的组件状态（default、hover、focus、active、disabled）
- 符合WCAG 2.1无障碍标准

## 4. 状态管理（优化对齐UI设计规范）
项目采用分离式状态管理策略，将UI状态和业务状态分别管理，确保与UI设计规范的状态管理要求完全对应。

### 4.1 UI状态 (React Context) - 界面交互状态
UI状态通过`UIContext`管理，主要负责界面展示相关的状态，特别是主题切换功能，严格对齐UI设计规范。

- **实现文件**：`/src/contexts/UIContext.tsx`
- **功能**：
  - **主题模式**：浅色/深色主题切换（支持system模式自动检测系统主题），严格遵循UI设计规范的OKLCH色彩空间
  - **界面交互状态**：侧边栏展开/收起、模态框显示/隐藏、下拉菜单状态
  - **全局UI配置**：语言设置（zh-CN/en-US）、响应式布局适配、无障碍设置
  - **临时UI状态**：加载状态、动画状态、通知管理、面包屑导航状态
  - **组件状态管理**：支持UI设计规范定义的所有组件状态（default、hover、focus、active、disabled）

**优化对齐UI设计规范**：
- 主题切换实现与UI设计规范的OKLCH色彩空间完全对应
- 组件状态管理支持UI设计规范定义的所有交互状态
- 响应式状态管理支持UI设计规范的断点系统

```typescript
// UIContext核心功能示例（优化对齐UI设计规范）
interface UIState {
  // 主题状态 - 严格对齐UI设计规范
  theme: ThemeType; // 'light' | 'dark' | 'system'
  
  // 界面交互状态
  sidebarOpen: boolean;
  modalOpen: boolean;
  dropdownOpen: boolean;
  
  // 全局UI配置
  language: 'zh-CN' | 'en-US';
  accessibility: {
    highContrast: boolean;
    reducedMotion: boolean;
  };
  
  // 临时UI状态
  loading: boolean;
  notifications: Notification[];
  breadcrumbs: BreadcrumbItem[];
}

// 主题切换实现 - 严格对齐UI设计规范
const applyThemeToDocument = useCallback((theme: ThemeType) => {
  // 根据主题类型设置类名
  const actualTheme = theme === 'system' ? getSystemTheme() : theme;
  
  // 添加Tailwind标准的dark类名，确保与UI设计规范一致
  if (actualTheme === 'dark') {
    document.documentElement.classList.add('dark');
    // 应用UI设计规范的深色主题OKLCH色彩
    document.documentElement.style.setProperty('--primary-color', 'oklch(65.2% 0.251 10)');
    document.documentElement.style.setProperty('--background-color', 'oklch(12.3% 0 0)');
  } else {
    document.documentElement.classList.remove('dark');
    // 应用UI设计规范的浅色主题OKLCH色彩
    document.documentElement.style.setProperty('--primary-color', 'oklch(65.2% 0.251 10)');
    document.documentElement.style.setProperty('--background-color', 'oklch(99.9% 0 0)');
  }
  
  // 保存主题设置到localStorage
  try {
    localStorage.setItem('app-theme', theme);
  } catch (error) {
    console.warn('Failed to save theme to localStorage:', error);
  }
}, [getSystemTheme]);
```

### 4.2 业务状态 (Redux Toolkit) - 应用数据状态

  - **UI业务状态**：页面级业务状态，如表单验证状态、筛选条件等
  - **异步操作**：
    - getBookings: 获取用户预约列表
    - getAvailableSlots: 获取指定日期的可用时间段
    - createBooking: 创建新预约
    - cancelBooking: 取消预约

**优化对齐UI设计规范**：
- 状态管理支持UI设计规范的数据展示需求
- 异步状态管理支持UI设计规范的加载状态和错误状态
- 表单状态管理支持UI设计规范的验证和反馈机制

- **功能**: 智能判断页面是否需要AppLayout布局
- **排除页面**: `/login`, `/register`, `/demo-ui` 保持独立布局
- **包含页面**: 其他所有页面自动应用AppLayout

```typescript
// PageWrapper组件核心实现
const PageWrapper = ({ Component, pageProps }: AppProps) => {
  const router = useRouter();
  const { currentUser } = useSelector((state: RootState) => state.user);
  
  // 不需要AppLayout的页面路径
  const noLayoutPages = ['/login', '/register', '/demo-ui'];
  
  // 检查当前页面是否需要AppLayout
  const needsLayout = !noLayoutPages.includes(router.pathname);
  
  // 如果页面需要布局包装
  if (needsLayout) {
    return (
      <AppLayout
        isLoggedIn={!!currentUser}
        onLogout={handleLogout}
      >
        <Component {...pageProps} />
      </AppLayout>
    );
  }
  
  // 否则直接渲染页面组件
  return <Component {...pageProps} />;
};
```

```typescript
const useResponsive = () => {
  const [breakpoint, setBreakpoint] = useState<'xs' | 'sm' | 'md' | 'lg' | 'xl' | '2xl'>('xs');

  useEffect(() => {
    const handleResize = () => {
      const width = window.innerWidth;
      if (width < 480) setBreakpoint('xs');
      else if (width < 640) setBreakpoint('sm');
      else if (width < 768) setBreakpoint('md');
      else if (width < 1024) setBreakpoint('lg');
      else if (width < 1280) setBreakpoint('xl');
      else setBreakpoint('2xl');
    };

    handleResize();
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return {
    breakpoint,
    isMobile: breakpoint === 'xs' || breakpoint === 'sm',
    isTablet: breakpoint === 'md',
    isDesktop: breakpoint === 'lg' || breakpoint === 'xl' || breakpoint === '2xl',
    isLargeScreen: breakpoint === 'xl' || breakpoint === '2xl',
  };
};
```
